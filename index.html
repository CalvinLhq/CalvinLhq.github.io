Vim 和 Tmux

一个一个说，先讲讲 Vim 编辑器。什么是 Vim 呢？如果你用过 Windows，那你一定用过 Word；如果你用过 Mac，那你一定听说过 Pages；如果你想要学 Linux，那你必须要掌握 Vim。Vim 就是个编辑文本的工具。
Tmux 是一个终端复用工具，这里有两个概念需鸟理解，终端 以及 复用。终端就是我们这黑乎乎的面板啦，我们在终端处输入命令，通过终端与系统交互。但是这样就有一个问题，我们每次只能做一件事情啊。举个例子，我们在使用 yum install 命令安装一个比较大的软件，在安装期间由于无法输入命令，我们就只能干等着。Tmux 使我们可以把一个终端一分为二，二分为三，完美的解决了终端不够用的问题，这就是 复用。
Tmux 还有一个优势，当我们在 ssh 远程连接服务器时，有时会因为网络连接或其他原因导致 会话 中断，用户与计算机的一次临时交互被称为 会话。由于窗口与会话和进程是紧密连接在一起的，所以会话的中断也会导致进程的停止。网络波动一下，工作成果全无？！这可不行，所以 Tmux 巧妙地使 会话 在我们退出 窗口 后持续运行。从此不用担心一不小心退出窗口导致会话中断了。

学习-Linux-基本操作

两眼抹黑啥也不知道，那就从安装系统开始吧！

  1. 安装 Linux 系统
    话说是要安装系统，但要如何开始呢？从哪里找来系统？找来什么系统？又要安装到哪里去呢？这些都是我要攻破的问题。

    • 安装 Linux 系统之前准备工作：
      对于新手来说，练习一般都是在个人电脑上完成的，但是直接在电脑上重装系统，将原有系统覆盖显然是行不通的。
      比较合理的方式是，在电脑上安装虚拟机环境，并在环境内练习安装 Linux 系统的虚拟机。

      • 安装 VMware Workstation 虚拟机
        步骤很简单，到 VMware 官网去下载就好了。
      • 下载合适的系统镜像
        什么是镜像？我们一般称其为光盘镜像。你可以把一个镜像理解成一张光盘，一张没有实体的以文件形式存在的光盘，里面封存着一个操作系统的所有信息。镜像可以从网上各种渠道下载，我从网上选择了 CentOS 7.6 版本的镜像。CentOS 7.6 是众多 Linux 发行的操作系统之一，容易上手，方便安装。有了镜像和 VMware Workstation，就可以开始安装了。

    • 开始安装
      在 VMware Workstation 内选择新建虚拟机，然后按照步骤和提示创建即可。创建完成后会开始初始化程序，黑底白字的命令行开始在眼前奔腾，不需要去顾虑上面的内容。等待一会后进入 CentOS 安装界面，大部分设置项都不用改动，其中值得一提的是硬盘的分配。

      • 硬盘分配
        这里我了解到了 Linux 系统万物皆文件的属性，稍微学习了一下各主要文件目录的作用。之后暂且根据网络上他人推荐的配置完成了设置。
      • 设置 root 用户密码
        root 用户是权限最高的管理员，所以不要忘记 root 用户的密码。忘记密码的话也有方法可以重置，方法繁琐，可以在网络上找到方法。设置完密码后点击安装就完成了。

    • 配置网络
      从这一步开始，难度开始上升。
      操作系统安装完毕之后以 root 用户登入到系统中，进入终端，也就是命令行，然后输入 ifconfig 或者 ip a 后回车来查看网卡配置情况。找到正在使用的网卡后，使用 vim 编辑器 修改 /etc/sysconfig/network-scripts/网卡名 路径下的文件。具体步骤在 虚拟化与远程连接 中会更详细的介绍。

  1. 学习 Linux 命令行指令
    一般情况下，我们的工作都是在命令行模式下进行的。可以使用 systemctl set-default multi-user.target 命令切换到命令行模式，这样卡机以后就能进入到黑底白字的命令行了。

    • 目录管理
      在 Linux 系统中工作必不可少的要在众多文件目录中穿梭。所以所以除了知晓主要目录的作用外，还要掌握如何创建，移动，命名，授权目录的方法。
    • 文件管理

以上都是最为基础的文件指令，相当于鼠标于图形化界面。Linux 学习是一个漫长的积累过程，设计者们也考虑到了指令繁多可能会导致的使用困难，于是也开发了一些帮助我们使用 Linux 的指令：想要了解的指令 --help 或 man 想要了解的指令。这两个指令会提供给我们想要了解的信息，前者提供的信息更精简，后者更详细。在学习期间要善于去运用这两个指令。
实用的命令还有太多太多，重启系统的 reboot，管理软件包的 tar 和 yum，用来查找文件的 find 等。都需要根据具体的工作内容进行调整学习。

我的第一篇博客

嗨！你好。这是我的第一篇博客。记录了我大一实习期间所学到的技术知识。本人水准不高，因此所分享的内容不甚专业，也不够深入，但绝对真挚。实习公司业务主要面向云计算虚拟化，所以分享的内容也会专注于 Linux 相关指令、工具、以及基于 Linux 系统的虚拟化技术。 最后，虽然这更偏向于一个个人总结性质的不成熟博客，也希望能多少帮助到正在读这段话的你。（会持续更新）

总览

一、在进入公司实习之前，我是一个纯纯的计算机小白。在计算机领域的至高成就除了一次给台式机重装 window 系统的经历，就是大一在学校学的两个多月 C 语言了。

  • 那我要学些什么呢？
    取决于公司的业务。

  • 公司的业务是什么呢？
    云计算服务为主。

  • 学云计算要学什么呢？
    那就多了，不过第一步，我准备学习 Linux 操作系统，因为我被告知服务器上的虚拟机都是运行在 Linux 系统之上的。

于是，画布上的第一笔决定了。 总之先学 Linux 。

二、稍微熟悉 Linux 命令行的操作后，作为实践学习，我开始在服务器上实际安装并配置 RedHat 发行版的 Linux 系统。当然此时我仍然是只知其然，不知其所以然的状态。正如欧洲人第一次见到筷子，即使告知其是餐具，也完全没有头绪该如何用两根棍吃饭一样。

Linux 和虚拟化有什么关系呢？ 我先暂时不去考虑这么深奥的东西，以目力所及的困难为优先，首先我要知道：

  • 哪些知识我不得不掌握？
    网络基础。

    在系统安装完成以后，每次必不可少的都要对网络进行配置。但 IP、NetMask、Gateway、DNS、DHCP、交换机、路由器 等等名词使我头晕眼花，给我当头一棒。

    于是，我在 Windows 笔记本上开始了针对网络的练习。 通过远程连接学习网络基础 .

三、Linux 和网络都略有了解后，开始了解虚拟化。

  • 什么是虚拟化？
    通过软件，将物理硬件资源进行再分配。方便管理。

  • 什么样的软件呢？
    虚拟化的软件有不少，比如 Qemu，比如 Libvirt，但要学习虚拟化进了解这些软件还不够！打个比方，对于一辆车来说发动机固然重要，但轮子方向盘等部件缺一不可啊，各个部件呆在适当的地方互相配合相得益彰，如此这般，才可真正投入使用。虚拟化在这个方面也是一样的，想要达成虚拟化，需要各个不同的组件互相配合。这里我了解到了 OpenStack，一台可以跑起来的合格的车。它提供了虚拟化以及其调度的全套解决方案，由许多各司其职的组件组合而成。

    于是，我开始 学习 OpenStack

四、逐渐的，通过学习，我开始理解自己在做什么。那么是时候开始加速融入工作环境了。

  • 一个开发团队该如何管理统筹所有的代码呢？
    每个公司可能有不同的方案，我则被要求学习 Git，一个分布式管理代码的解决方案。

    于是，我开始 学习 Git

五、此时，时间已经过去了一个多月，实习还剩大概相同的时间。至此，更多的需要知识支持的工作被分配给了我。实质性的工作使我信心大增，同时也不能忘了自我提升。

  • 还有什么可学的吗?
    有啊，多的是。基于 Linux 的开源属性，数不清的工具和插件被开发出来，我也针对实际需求去学习了两个使用的 Linux 工具–Tmux 终端复用工具和 Vim 编辑器，掌握前者可使 Linux 终端操作事半功倍，后者更是 Linux 环境下工作的必不可缺。

    于是，我开始学习 Linux 实用工具 Vim 和 Tmux

六、随着时代的发展，技术的更新迭代与淘汰，OpenStack 虚拟化技术也被发现在个别场景略显笨拙。

  • 什么叫略显笨拙？
    大部分情况下，一个服务无法将一台虚拟机的资源物尽其用，再加上操作系统本身就会占用很多资源，所以就会有大材小用、头重脚轻的既视感。

  • 解决方案是？
    这就必须提到 Docker，或者说容器化技术了，一个比虚拟机更轻量化虚拟化解决方案。

    于是，我花了几天去 学习 Docker 的使用方式

七、这三个月学到了很多，雪白的画布上被勾勒出了清晰可见的轮廓。但仍需有一个地方将这些知识储存起来，使得我能随时预览，随时改进，随时添加。

  • 以什么方式对一段时间的学习进行收尾比较好呢？
    当然是写总结了。

    于是，我写了这篇博客。想知道如何建立自己的博客？可以看看 我是怎么写博客的
