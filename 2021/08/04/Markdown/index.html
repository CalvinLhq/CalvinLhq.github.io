Markdown

你可以将 Markdown 理解为一个标准，一个规则。它规定了用户以什么样的格式编写文本会得到怎样的效果。通常，以 Mardown 格式编写的文本会被冠以 .md 后缀。
如果你阅览过一定数量的博客的话，会发现文章的格式会有一种既视感。好像都不无相似，而且都简洁直观：加粗的黑色标题，分层级列表，规范的代码块等等。当然，我的所有博客也都是用 Markdown 写的。
下面，我将以一篇范例的形式来帮助你理解 Markdown 的编写方法。与文章后半代码块部分对照食用更加哦。

一、标题一

在标题前加上一个井号键并空格

标题二

每多加一个井号，标题尺寸就缩小一点

标题三

以此类推

标题四

还会变小

标题五

到六个井号键为止

标题六

二、分割线

通常情况下标题二下方都会有一条分割线，想要添加分割线可以在单独的一行输入三个减号 — 即可。

注意分割线要与不能与上一行文本相接，要空一行，不然就会这样，变成标题

三、列表嵌套

  • 第一层，文本之前加上一个 - 还有一个空格
    • 第二层，文本之前按一下 tab 键，加上一个 - 还有一个空格
      • 第三层，两个 tab 键，将上一个 - 还有一个空格
        • 第四层，以此类推
          • 五层极限
          • 第五层，同级列表
      • 第三层，统计列表
  1. 序号一，数字列表，文本之前加上数字和英文模式下的逗号 . 还有一个空格。
    • 第二层
  2. 序号二
    • 第二层

四、换行

这部分内容可能有点小麻烦，
需要对代码块中的内容进行对照学习。
如何进行换行呢？最开始的一句话其实在编写时于逗号处使用了一次 enter 键，按理来说应该是换行了呀？实际上使用markdown格式编写换行时需要先点两下空格（这后面有两个空格）
再点 enter 键，于是就可以在下一行编写内容了。

空行很简单，点两次 enter 空出一行就可以了。

五、链接和图片

图片和连接的引用方式很相似

  • 图片：![图片描述文字](图片地址 "图片名")
  • 链接：[描述名称](链接地址)

六、表格

一 二 三
内容一 内容二 内容三
四 五 六
  • 一般情况下在第二行的分隔符 - 左边加 : 会使此列文字靠左，右边加 : 则靠右，两边都加则居中.

七、代码块

代码块可以将代码库框住，使其更直观显眼。框选方式分为两种：

  • 第一种针对在行内出现的个别代码，我们使用反引号，键盘上 esc 下面的波浪号那个键，将代码围住 这是代码。
  • 另外一种就是单独的大型代码块，在代码块开头处和结尾处各使用三个反引号，将代码包围住。以下我将本片文章用代码块包住，使原本的格式展现出来，方便对照学习：
