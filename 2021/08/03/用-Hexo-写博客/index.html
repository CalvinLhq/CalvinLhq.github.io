用 Hexo 写博客

有入口就一定有出口，输入的知识则注定被输出。道理谁都懂，那么就从现在开始向自己的博客上疯狂输出吧！将自己的心得体会，无处拖放的奇思妙想输送到互联网这个连接着全世界人的网上。没准火了呢？

这里选择的博客生成器叫 Hexo，一名很 kind 的同事推荐给我的。他也拿 Hexo 建立过自己的博客，感兴趣的同学可以去看看：https://blog.tachao.top/
那么我们就开始搭建自己的博客环境吧！

首先，我的一切操作都完成于 Linux 系统之下，对 Linux 还不够熟悉的可以先去看一看我关于 初学 Linux 的经历 。而且需要掌握 Vim 编辑器， Git 和 Markdown 格式编写方法 等基本知识.

  1. 安装前步骤：
    • 首先在官网上下载 nodejs 的 tar 包至 Linux 虚拟机上
    • 使用 tar -xzf 将其解压
    • vim /etc/profile 配置环境变量并使用 source /etc/profile 使其生效
      • 在文件末尾添加以下代码：
    • 使用 npm -v node -v 查看 node 和 npm 版本并确认安装成功
  2. 可以开始安装了：
    • 使用 npm install -g hexo-cli 下载安装 hexo
    • 安装完以后初始化：
      • 创建一个空的 blog 目录，cd 到目录下执行 hexo init 进行初始化，稍候片刻会多出来一堆配置文件（_config.yml文件很重要，会经常用到）
  3. 创建文章准备发布：
    • 使用 hexo new post 创建第一篇文章,文章默认存放在 blog/source/_posts/ 目录下
    • 编辑完以后返回 blog 目录，使用 hexo clean 清理缓存先前的静态文件 (每次编辑完要再发布之前都要先删除已有静态文件，再生成新的静态文件)
    • 使用 hexo generate 去生成新的静态文件
  4. 准备 pages 环境：
    • 在 GitHub 上新建 repository 并命名为 用户名.github.io
    • 在 Linux 环境创建的 blog 目录下 安装 git 部署插件：
      • 使用 npm install --save hexo-deployer-git
