Vim 和 Tmux

一个一个说，先讲讲 Vim 编辑器。什么是 Vim 呢？如果你用过 Windows，那你一定用过 Word；如果你用过 Mac，那你一定听说过 Pages；如果你想要学 Linux，那你必须要掌握 Vim。Vim 就是个编辑文本的工具。
Tmux 是一个终端复用工具，这里有两个概念需鸟理解，终端 以及 复用。终端就是我们这黑乎乎的面板啦，我们在终端处输入命令，通过终端与系统交互。但是这样就有一个问题，我们每次只能做一件事情啊。举个例子，我们在使用 yum install 命令安装一个比较大的软件，在安装期间由于无法输入命令，我们就只能干等着。Tmux 使我们可以把一个终端一分为二，二分为三，完美的解决了终端不够用的问题，这就是 复用。
Tmux 还有一个优势，当我们在 ssh 远程连接服务器时，有时会因为网络连接或其他原因导致 会话 中断，用户与计算机的一次临时交互被称为 会话。由于窗口与会话和进程是紧密连接在一起的，所以会话的中断也会导致进程的停止。网络波动一下，工作成果全无？！这可不行，所以 Tmux 巧妙地使 会话 在我们退出 窗口 后持续运行。从此不用担心一不小心退出窗口导致会话中断了。
